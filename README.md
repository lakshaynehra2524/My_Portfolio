# Lakshay — Portfolio

A personal portfolio site built as a FastAPI app: a single-page site styled
around a computer-vision "detection/annotation" visual language (bounding-box
brackets, confidence-score tags, status badges), backed by a couple of tiny
API endpoints so LinkedIn posts and GitHub repos can be added or edited
without touching code.

**Live sections:** Hero · About · Skills · Experience · LinkedIn · GitHub ·
Projects · Education · Contact

---

## Features

- **FastAPI backend** serving the static site plus a small JSON-backed API.
- **LinkedIn posts feed** — add a post link, title, date, and snippet from
  `/admin`; it shows up on the site immediately in a compact horizontal
  scroll strip.
- **GitHub repos feed** — same idea, but with room for a longer description,
  a status badge (active / paused / completed), a progress bar for active
  repos, tags, and a last-updated date.
- **Edit in place** — both the LinkedIn and GitHub sections support editing
  an existing entry from `/admin`, not just add/delete.
- **Profile photo upload** — swap your photo from `/admin`; it renders in an
  animated, organically-shaped frame on the hero.
- **Liquid background animation** — a canvas-based fluid/blob effect that
  reacts to cursor movement, independent of clicks so it never interferes
  with reading or clicking the page content.
- **Simple key-based auth** — every write endpoint (`POST`/`PUT`/`DELETE`)
  requires an `X-Admin-Key` header matching your `PORTFOLIO_ADMIN_KEY`.

## Project structure

```
main.py                FastAPI app — serves the site + all /api/* routes
requirements.txt
data/
  posts.json            LinkedIn post entries (seed data — replace via /admin)
  repos.json             GitHub repo entries (seed data — replace via /admin)
  profile.json             created automatically after your first photo upload
static/
  index.html              the portfolio itself
  admin.html                the /admin page (add / edit / delete posts & repos, upload photo)
  uploads/                   your uploaded profile photo lands here
.gitignore
```

## Run it locally

```bash
pip install -r requirements.txt
export PORTFOLIO_ADMIN_KEY="pick-a-real-secret-here"
uvicorn main:app --reload
```

| URL | What it is |
|---|---|
| `http://127.0.0.1:8000/` | the portfolio |
| `http://127.0.0.1:8000/admin` | add/edit LinkedIn posts, GitHub repos, and your photo |
| `http://127.0.0.1:8000/docs` | interactive API docs (auto-generated by FastAPI) |

## API reference

All write routes require an `X-Admin-Key` header.

| Method | Route | Purpose |
|---|---|---|
| GET | `/api/posts` | list LinkedIn posts |
| POST | `/api/posts` | add a post |
| PUT | `/api/posts/{index}` | update a post |
| DELETE | `/api/posts/{index}` | remove a post |
| GET | `/api/repos` | list GitHub repos |
| POST | `/api/repos` | add a repo |
| PUT | `/api/repos/{index}` | update a repo |
| DELETE | `/api/repos/{index}` | remove a repo |
| GET | `/api/profile` | get profile photo URL |
| POST | `/api/profile/photo` | upload/replace profile photo |

Indexes are positions in the current list, not stable IDs — re-fetch before
editing/deleting if you're scripting against the API directly rather than
using `/admin`.

## Deploying

Any host that runs a Python process works — Render, Railway, Fly.io, a
plain VPS, etc.

**Start command:**
```bash
uvicorn main:app --host 0.0.0.0 --port $PORT
```

**Before you deploy:**

1. Set a real `PORTFOLIO_ADMIN_KEY` environment variable on your host —
   don't ship default.
2. Make sure your host's filesystem is persistent across deploys/restarts.
   `data/posts.json`, `data/repos.json`, `data/profile.json`, and your
   uploaded photo are plain files on disk — some free-tier hosts wipe these
   on every redeploy. If yours does, either pick a host with a persistent
   volume, or move photo storage to an object store (S3, Cloudinary, etc.)
   instead of local disk.
3. Swap the placeholder GitHub repo URLs in `data/repos.json` (or via
   `/admin`) for your real ones.
4. Add your real photo and first LinkedIn post/repo via `/admin` once it's
   live — or edit `data/posts.json` / `data/repos.json` before your first
   deploy so the site isn't empty on day one.
5. `.gitignore` already excludes `data/profile.json`, `static/uploads/*`,
   and any `.env` file — see that file for the full list.

## About the LinkedIn integration

There's no free way to make LinkedIn auto-push your posts to a personal
site — their API only opens that door to approved business/Partner Program
integrations, and profile photos aren't hotlinkable outside their app
either. This project's `/admin` page is the practical alternative: adding
or editing a post takes about ten seconds, and it renders properly styled
instead of an embedded iframe.
